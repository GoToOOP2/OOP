| 항목 | 내용                                                                                      |
|---|-----------------------------------------------------------------------------------------|
| 점검 목적 | 동시성 문제 유발 가능성이 있는 가변 인스턴스 변수 식별 (즉 가변 을 찾는거임)                                           |

## 위험 판단 기준 근거

Spring Bean(`@Service`, `@Repository`, `@RestController`)은 싱글톤으로 관리된다. 즉, 인스턴스가 딱 하나이고 모든 요청(스레드)이 그 인스턴스를 공유한다.

따라서 그 인스턴스의 필드도 함께 공유되는데, 여러 스레드가 동시에 읽고 쓰면 Race Condition이 발생한다.


**위험하다고 보는 것**

- `var` 필드 — 재할당 가능하므로 여러 스레드가 동시에 write하면 값이 꼬임
- `val`이어도 `MutableList`, `MutableMap` 등 가변 컬렉션 — 참조는 고정이어도 내부 상태(`add()`, `remove()`)는 변경 가능

**제외하는 것**

- `val` + 불변 타입(`String`, `Int` 등) — 재할당도, 내부 변경도 불가능
- 생성자 주입 의존성 — Spring이 초기화 시점에 한 번만 세팅하고 이후 변경 없음

---

## 위험도 기준

| 위험도 | 설명 |
|---|---|
| 높음 | 여러 스레드에서 동시에 write 가능성이 명확한 가변 필드 |
| 중간 | 가변 필드이나 write 경로가 제한적인 경우 |
| 낮음 | 가변 선언이지만 실질적 위험이 낮은 경우 |
